## Пакет src

Пакет содержит набор модулей для работы с клиентскими данными и операциями

### Модуль masks

Модуль содержит функции, позволяющие маскировать данные пользователя

#### Функция get_mask_card_number(card_number: int | str) -> str

Функция принимает на вход номер карты и возвращает ее маску

**Пример входных данных**: `1234567891234567`

**Пример выходных данных**: `1234 56** **** 4567`

При передаче в функцию некорректного номера карты, функция вернет исключение 

`ValueError("Incorrect card number")`

#### Функция get_mask_account(account: int | str) -> str

Функция принимает на вход номер счета и возвращает его маску

**Пример входных данных**: `1234567891234567`

**Пример выходных данных**: `**4567`

При передаче в функцию некорректного номера счета, функция вернет исключение 

`ValueError("Incorrect account number")`

### Модуль widgets

Модуль содержит функции, позволяющие обрабатывать данные пользователя

#### Функция mask_account_card(card_account_number: str) -> str

Принимает строку, содержащую тип и номер карты или счета.
Возвращает строку с замаскированным номером.
Для маскировки номера карты и номера счета используются разные маски.

**Пример входных данных для карты**: `Visa Platinum 7000792289606361`

**Пример выходных данных для карты**: `Visa Platinum 7000 79** **** 6361`

**Пример входных данных для счета**: `Счет 73654108430135874305`

**Пример выходных данных для счета**: `Счет **4305`

При передаче в функцию некорректных входных данных, функция вернет исключение:

`ValueError("Incorrect input data")`

#### Функция get_date(date_string: str) -> str

Принимает на вход дату в формате ISO строки, отдает дату в формате ДД.ММ.ГГГГ

**Пример входных данных**: `2024-03-11T02:26:18.671407`
**Пример выходных данных**: `11.03.2024`

При передаче в функцию некорректных входных данных, функция вернет исключение:

`ValueError("Incorrect input data")`

### Модуль processing

Модуль содержит функции, позволяющие работать со списком операций пользователя

#### Функция filter_by_state(operations_list: List[dict], state: str = "EXECUTED") -> List[dict]

Принимает на вход список словарей с данными об операциях и статус операции.
Значение статуса по умолчанию 'EXECUTED'.
Возвращает список операций с запрашиваемым статусом.

**Пример входных данных**: 
```
[
	{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}, 
	{'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'}, 
	{'id': 594226727, 'state': 'CANCELED', 'date': '2018-09-12T21:27:25.241689'}, 
	{'id': 615064591, 'state': 'CANCELED', 'date': '2018-10-14T08:21:33.419441'}
]
```
**Пример выходных данных для статуса по умолчанию 'EXECUTED'**: 
```
[
	{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}, 
	{'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'}
]
```
**Пример выходных данных для статуса 'CANCELED'**: 
```
[ 
	{'id': 594226727, 'state': 'CANCELED', 'date': '2018-09-12T21:27:25.241689'}, 
	{'id': 615064591, 'state': 'CANCELED', 'date': '2018-10-14T08:21:33.419441'}
]
```

#### Функция sort_by_date(operations_list: List[dict], sort_reverse: bool = True) -> List[dict]

Функция получает на вход список операций и сортирует их по дате в указанном направлении.
По умолчанию сортировка происходит по убыванию

**Пример входных данных**: 
```
[
	{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}, 
	{'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'}, 
	{'id': 594226727, 'state': 'CANCELED', 'date': '2018-09-12T21:27:25.241689'}, 
	{'id': 615064591, 'state': 'CANCELED', 'date': '2018-10-14T08:21:33.419441'}
]
```
**Пример выходных данных для сортировки по умолчанию (True)**: 
```
[
	{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}, 
	{'id': 615064591, 'state': 'CANCELED', 'date': '2018-10-14T08:21:33.419441'},
	{'id': 594226727, 'state': 'CANCELED', 'date': '2018-09-12T21:27:25.241689'}, 
	{'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'}
]
```
**Пример выходных данных для сортировки по возрастанию (False)**: 
```
[ 
	{'id': 939719570, 'state': 'EXECUTED', 'date': '2018-06-30T02:08:58.425572'},
	{'id': 594226727, 'state': 'CANCELED', 'date': '2018-09-12T21:27:25.241689'}, 
	{'id': 615064591, 'state': 'CANCELED', 'date': '2018-10-14T08:21:33.419441'},
	{'id': 41428829, 'state': 'EXECUTED', 'date': '2019-07-03T18:35:29.512364'}
]
```
Если хотя бы одна операция содержит некорректную дату, функция вернет исключение

`ValueError("Incorrect operation date")`

### Модуль generators

Модуль содержит функции, реализующие генераторы для обработки данных

#### Функция filter_by_currency(transactions: list[dict], currency: str) -> Iterator | Generator

Принимает на вход список словарей, представляющих транзакции.
Возвращает итератор, который поочередно выдает транзакции,
где валюта операции соответствует заданной (например, USD).

**Пример входных данных**:
```
    [
        {
            "id": 939719570,
            "state": "EXECUTED",
            "date": "2018-06-30T02:08:58.425572",
            "operationAmount": {"amount": "9824.07", "currency": {"name": "RUR", "code": "RUR"}},
            "description": "Перевод организации",
            "from": "Счет 75106830613657916952",
            "to": "Счет 11776614605963066702",
        },
        {
            "id": 142264268,
            "state": "EXECUTED",
            "date": "2019-04-04T23:20:05.206878",
            "operationAmount": {"amount": "79114.93", "currency": {"name": "USD", "code": "USD"}},
            "description": "Перевод со счета на счет",
            "from": "Счет 19708645243227258542",
            "to": "Счет 75651667383060284188",
        }
    ]
```
**При первом вызове итератора будет возвращено значение**:
```
        {
            "id": 142264268,
            "state": "EXECUTED",
            "date": "2019-04-04T23:20:05.206878",
            "operationAmount": {"amount": "79114.93", "currency": {"name": "USD", "code": "USD"}},
            "description": "Перевод со счета на счет",
            "from": "Счет 19708645243227258542",
            "to": "Счет 75651667383060284188",
        }
```
При повторном - исключение

`StopIteration`

#### Функция card_number_generator(start: int, stop: int) -> Generator

Принимает на вход диапазон для генерации номеров карт и последовательно выдает номера в заданном диапазоне.

Если начало диапазона превышает окончание, то границы автоматически поменяются местами

**Примеры**:
```
for card_number in card_number_generator(1, 5):
    print(card_number)

>>> 0000 0000 0000 0001
    0000 0000 0000 0002
    0000 0000 0000 0003
    0000 0000 0000 0004
    0000 0000 0000 0005
```
```
for card_number in card_number_generator(5, 3):
    print(card_number)

>>> 0000 0000 0000 0003
    0000 0000 0000 0004
    0000 0000 0000 0005
```
При попытке запросить следующее значение генератора, вернется исключение

`StopIteration`
